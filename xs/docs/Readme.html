XS automatic editor and serialization for hierarchical Scala structures

XS is a library that generates an professional HTML5 editor and XML serialization for many Scala classes using reflection and a small amount of annotations.

Why you would want to use it

  • Professional editor with few annotations
  • Hierarchical structures deal with trees and/or tables
  • Powerful error reporting - squiggly underlines under the offending characters of fields with tooltips
  • Simple cases are easy, and many complex cases are possible
  • Can be used with a variety of front ends, including Java Servlets and Play
  • Drag and drop, cut and paste
  • Fields can be text, multiple choice, boolean, numbers, images, or anything that can be converted to/from strings and edited as strings
  • Internationalization using .properties files
  • Deals reasonably well with high latency connections

What does it require?

  • Scala 2.10 (uses reflection API)
  • jQuery (tested with 1.9, probably works with significantly earlier versions)
  • Slick.Grid if using grids
  • A few other jquery plugins
  • IE9 or later, recent Firefox, others under testing
  • Data structures that can be created with a constructor (rather than individually setting var fields)

Why would you not want to use it?

  • You need to support old browsers. Sorry.
  • You want to edit circular data structures.
  • Given a field of type Option[String], you want the user to be able to enter (and distinguish) None, null, Some(null), and Some(""). (many people would consider you mad in this case anyway)
  • You are very bandwidth constrained. XS sends a network packet each time the user does something (eg types a key). This enables dynamic error highlighting, but could add up to large mobile connection costs.
  • You want a stateless system that can scale to many servers easily. XS keeps track of what is being shown on a user's display in order to optimize updating. But this requires session tracking with significant state.

Documentation

There are conceptual parts of XS that the user deals with: the data model that describes the structures being edited, and the deployment environment which is how you fit XS into a web app.

How to use the UI and what it looks like.

There is a 5:57 video demonstrating some of the features. For a hands on experience, download and run the demo (see the deployment section)

Data model - annotations on your data structures

There is a ten slide PowerPoint presentation that gives a feeling for the data model. There is a more complete text introduction to most of the features. Javadocs/Scaladocs in the source code cover the minutiae of various annotations. More detailed information about Icons and text (localization) are available. Lastly there is a 13:11 video describing the data model behind the earlier video.

A fairly sophisticated examples are given in the org.greatcactus.xs.test package - start at SpaceClasses.scala
